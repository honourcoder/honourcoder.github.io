Mac中常用的GCC文件编译指令

  • C语言初级
  • GCC
必修课

GCC的意思也只是GNU C Compiler而已。经过了这么多年的发展，GCC已经不仅仅能支持C语言；它现在还支持Ada语言、C++语言、Java语言、Objective-C语言、Pascal语言等等很多语言.他是*inux系统中默认继承安装的.他的本意是让GNU系统彻底的自由软件,即用户有充分的软件自由.

GCC是非常好用和便利的,但是,对于我们来说,最重要的还是如何去使用GCC.

常用的GCC命令

准备工作


首先先准备好我们要用来编译运行的文件.
使用touch命令创建一个test.c文件.

touch test.c

然后使用vi命令编辑test.c文件.

vi test.c

在控制台输入下面的代码

#include < stdio.h >
int main(void)
{
    printf(“Hello World!\n”);
    return 0;
}

最后,我们按下Esc,然后再”:wq”保存退出.这里我们的准备工作就进行完了.

一次完成编译文件


GCC的编译指令的是

gcc test.c -o test

这条指令的作用是将test.c 直接编译成可执行文件.,结果输出到当前目录下”test”这个文件中.

如果想执行,在控制台输入下面的指令即可.

./test

这个时候,如果终端输出hello world,就证明你成功了.

分步完成编译


上面的指令是一次将整个文件都编译完成.其实整个的编译过程是分为四个步骤的:

1 预处理(也称预编译，Preprocessing)

2 编译(Compilation)

3 汇编(Assembly)

4 连接(Linking)。

很幸运的是,强大的GCC支持对这四步单步编译的.

执行预处理


GCC支持编译到预处理阶段完成,即,一些预处理的操作在这里都会被完成,最常见的就是,宏定义会在这个阶段完成全部的替换.可以利用这个阶段来清楚的看到宏定义的替换.
编译指令为:

gcc -E test.c -o test.i
或
gcc -E test.c

第一种方式是指定了输出文件,是将输出的结果存在指定的文件中.如果不指定,那么直接将编译的结果打印在控制台

执行编译处理


预处理之后，可直接对生成的test.i文件编译，生成汇编代码：

gcc -S test.i -o test.s

gcc的-S选项，表示在程序编译期间，在生成汇编代码后，停止，-o输出汇编代码文件。注意这里的S是大写.

执行汇编处理


对于上一步中生成的汇编代码文件test.s，gas汇编器负责将其编译为目标文件，如下：

gcc -c test.s -o test.o

执行连接处理


gcc连接器是gas提供的，负责将程序的目标文件与所需的所有附加的目标文件连接起来，最终生成可执行文件。附加的目标文件包括静态连接库和动态连接库。

对于上一步中生成的test.o，将其与Ｃ标准输入输出库进行连接，最终生成程序test

gcc test.o -o test

执行可执行文件


在控制台输入下面的指令即可.

./test

这个时候,如果终端输出hello world,就证明你成功了.
